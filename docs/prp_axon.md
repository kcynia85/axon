### 🚨 CRITICAL FRONTEND PROTOCOLS (Mandatory)
> **Ref:** `docs/knowledge/tech/stack-react-ts.md`, `docs/knowledge/structure/modular-monolith-nextjs.md` & `docs/product/planned_changes/Open_Source_React_Libraries.md`

1.  **Modular Structure (Standard):**
    *   **Root:** `src/modules/[module_name]/`.
    *   **Shared Kernel:** `src/shared/{domain,ui,lib}`.
    *   **Module Layers (Vertical Slice Architecture):**
        *   `domain/` (Shared Domain: Entities, Errors, Types - No React).
        *   `shared/` (Module-internal shared DTOs/mappers).
        *   `features/[feature-name]/` (The Vertical Slice):
            *   `ui/` (Components, Views - imports `application`).
            *   `application/` (Hooks, Use Cases, State - imports `domain` & `infrastructure`).
            *   `infrastructure/` (API calls, server actions).
    *   **Public API:** Cross-module imports MUST use `index.ts` barrel files.
2.  **React Patterns:**
    *   **No Classes:** Use Functional Components and Hooks only.
    *   **Data Fetching (Standard):** 
        *   **Strategy:** One query to fetch full list (`staleTime: 5min`) + Local state filtering (Client-side) for instant feedback.
        *   **Tool:** `TanStack Query`. **NEVER use `useEffect` for fetching.**
    *   **State:** Local state via `useState/useReducer`. Global state via `Zustand`.
    *   **Forms:** `React Hook Form` + `Zod`.
    *   **Auto-Save:** Universal `useAutoSave` hook pattern for all creative/config forms (Agents, Crews, Canvas).
    *   **Perf:** Default to Server Components (RSC). Use `"use client"` only for interactive leaves.
3.  **UI Libraries & UX:**
    *   **Core:** Tailwind CSS + Shadcn/UI.
    *   **Canvas:** **React Flow**.
    *   **Data Tables:** **TanStack Table**.
    *   **Drag & Drop:** **dnd-kit**.
    *   **Editor:** **BlockNote**.
    *   **Fonts:** Geist Mono.
    *   **Layouts:** Split Layout (List + Inspector/Preview) as default for Resources.
4.  **Code Quality:**
    *   **DDD Naming:** Use descriptive names (Ubiquitous Language). Code must read like a book.
    *   **Immutability:** Never mutate data. Use `const`.
    *   **Validation:** All inputs/forms/API responses must be validated with **Zod**.
    *   **Naming:** PascalCase for Components, camelCase for functions/variables.
    *   **Limits:** Max 500 lines per file. Split if larger.
5.  **Testing Strategy:**
    *   **Unit:** Jest/Vitest for pure logic and components.
    *   **Integration:** Test module interactions.
    *   **E2E:** Critical flows (Playwright/Cypress).
    *   **Rule:** No PR merge without passing tests.

### 6. Offline Mock Protocol (Resilience)
*   **Principle:** Frontend development must not be blocked by Backend availability.
*   **Implementation:** All feature `infrastructure` layers must implement a `USE_MOCK` toggle.
*   **Standard:**
    *   Create `mock-api.ts` alongside real `api.ts`.
    *   Use `shared/lib/mock-delay` to simulate network latency (300-800ms).
    *   Mock data must respect the Domain Entity types (Zod/TS).

## 2. Entities & Domain Vocabulary
### Core Entities
*   **Project (Aggregate):** The central container for user work. Groups Artifacts, Key Resources and links to a Space.
    *   *CodeName:* `Project`.
*   **Key Resource (Entity):** External link (Notion, Figma, Github) attached to a Project.
    *   *CodeName:* `KeyResource`.
*   **Artifact (Entity):** A tangible output generated by an Agent/Crew (Draft/In Review/Approved).
    *   *CodeName:* `Artifact`. Fields: `artifact_source_path`, `artifact_deliverable_url`, `artifact_approval_status`, `#approved_by_user_id`, `artifact_approved_at`, `workspace_domain`.
*   **Space (Aggregate):** A visual workspace (Canvas) containing Nodes (Agents, Crews, Notes) and Zones. Can be standalone or linked to a Project.
    *   *CodeName:* `Space`.
*   **SpaceZone (Entity):** A visual area on the Canvas linked to a Workspace domain.
*   **SpaceNode (Entity):** A node on the Canvas representing a specific component (Agent, Crew, etc.).
    *   Execution states: `Idle | Missing Context | Working | Briefing | Consultation | Done | Error`.
*   **NodeEdge (Value Object):** A connection between two SpaceNodes.
*   **ExecutionLog (Entity):** A trace of node execution status, outcomes, streaming output, token/cost metrics.
*   **ChatSession (Aggregate):** A consultation session between user and agent (Consultation mode).
*   **Workspace (Value Object/Zone):** A logical domain bucket (Product, Discovery, Design, Delivery, Growth). Visualized as "Zones" on Canvas.

### Workspace-Scoped Entities (each has `availability_workspace: List<Enum>`)
*   **Agent (Entity):** An autonomous actor with Role, Goal, Backstory, Guardrails, Tools, RAG config, I/O Schema, Temperature, Few-Shot, Reflexion.
    *   *CodeName:* `AgentConfig`. Links: `#assigned_tools_ids (N:M)`, `#llm_model_id (N:1)`, `#knowledge_hub_ids`.
*   **Crew (Entity):** A team of Agents with a Process (Sequential, Hierarchical, **Parallel/Consensus**).
    *   *CodeName:* `CrewConfig`. Links: `#agent_member_ids (N:M)`, `#manager_agent_id (N:1)` (hierarchical only).
*   **Pattern (Entity):** A reusable graph structure (saved from Canvas selection) with OKR context.
    *   *CodeName:* `Pattern`. Fields: `pattern_graph_structure (JSONB)`, `pattern_type`, `pattern_okr_context`.
*   **Template (Entity):** A manual checklist template with markdown content and to-do items.
    *   *CodeName:* `Template`. Fields: `template_markdown_content`, `template_checklist_items (JSONB Array)`.
*   **Prompt Archetype (Entity):** A reusable persona definition (Role/Goal/Backstory/Guardrails/Hubs) loadable into Agents.
    *   *CodeName:* `PromptArchetype`. Source: `axon_cm_recources.pdf`.
*   **Automation (Entity):** A webhook-based external process (n8n, Zapier, Make, Custom) with I/O schemas and validation.
    *   *CodeName:* `Automation`. Fields: `automation_platform`, `automation_http_method`, `automation_auth_config (JSONB)`, `automation_validation_status`, `automation_input_schema`, `automation_output_schema`.
*   **External Service (Entity):** External SaaS/tool registration (Figma, ElevenLabs) with capabilities.
    *   *CodeName:* `ExternalService`. Fields: `service_category (Utility|GenAI|…)`, `service_url`.
*   **Service Capability (Entity):** Specific capability offered by an External Service.
    *   *CodeName:* `ServiceCapability`. Links: `#external_service_id (N:1)`.
*   **Internal Tool (Entity):** Python `@tool` function synced from codebase. UI is read-only for definition, write-only for metadata.
    *   *CodeName:* `InternalTool`. Fields: `tool_function_name`, `tool_category`, `tool_input_schema`, `tool_output_schema`, `tool_is_active`.

### Knowledge Entities
*   **Knowledge Hub (Entity):** A folder/grouping for knowledge sources.
    *   *CodeName:* `KnowledgeHub`. Fields: `hub_name`, `hub_description`, `hub_keywords`, `hub_indexed_sources_count` (computed), `hub_last_source_indexed_at` (computed).
*   **Knowledge Source (Entity):** A file/URL in the Knowledge Base with processing metadata.
    *   *CodeName:* `KnowledgeSource`. Fields: `source_file_name`, `source_file_format`, `source_file_size_bytes`, `source_metadata (JSONB)`, `source_rag_indexing_status`, `source_chunking_strategy_ref`, `source_indexing_error`.
*   **Text Chunk (Entity):** A processed fragment of a knowledge source with embedding.
    *   *CodeName:* `TextChunk`. Fields: `chunk_index`, `chunk_text`, `chunk_embedding (vector)`, `chunk_metadata (JSON)`, `chunk_token_count`.

### Settings Entities
*   **LLM Provider (Entity):** Cloud/Meta-Provider/Local with API endpoint, key, tokenization strategy.
*   **LLM Model (Entity):** Model registry entry with capabilities, pricing, thinking support.
*   **LLM Router (Entity):** Aliased routing strategy (Fallback/Load Balancer) with primary + fallback model.
*   **Embedding Model (Entity):** Embedding provider config with dimensions, cost, max context.
*   **Chunking Strategy (Entity):** Named splitting method (Recursive Character, Code Splitter, Token Splitter) with params.
*   **Vector Database (Entity):** Vector store connection config (pgvector, ChromaDB, Pinecone) with index method.

### System-Level Entities
*   **Meta Agent (Entity):** Global Axon system assistant config (system prompt, temperature, RAG scope, LLM model).
*   **Voice Meta Agent (Entity):** Voice configuration for Meta Agent (provider: ElevenLabs/Azure/Google, voice_id).
*   **Inbox Item (Entity):** Notification entry (artifact_ready, consultation, approval_needed) linked to Artifact/Project.
    *   *CodeName:* `InboxItem`. Fields: `item_status (New|Resolved)`, `item_type`, `#artefact_source_id`, `#project_id`.
*   **User (Entity):** User profile with preferences, role (Admin/Worker).

### Key Actions
*   **Orchestrate:** The act of crewAI managing Task delegation between Agents.
*   **Ingest:** The ETL pipeline processing files/URLs into Knowledge Chunks.
*   **Briefing:** A specific Agent/Crew state where the plan is proposed and waits for user approval (Cost/Plan check).
*   **Consultation:** Human-in-the-loop interaction during execution — agent asks user a question.
*   **Sync Tools:** Updating Internal Tools definitions from Python code (`@tool` decorator) to the Database.

## 3. System Rules (Invariants)
> **Note:** These are immutable system rules that must be adhered to throughout the implementation.

1.  **Trustworthy Attribution:** Every answer derived from RAG MUST include inline citations (`[1]`, `[Source]`) pointing to specific chunks.
2.  **Project Isolation:** Users/Organizations can ONLY access data where they have permissions (enforced via Supabase RLS).
3.  **Unified System Awareness:** All system entities (Agents, Crews, Tools, Automations) MUST be indexed in `system_embeddings` table to allow Meta-Agents to "see" and use them.
4.  **No-UI Mutation for Internal Tools:** Internal Tools are defined in Code (Python) and synced to DB. UI is Read-Only for definition, Write-Only for metadata (tags/description).
5.  **Safe Delete:** Deleting an entity (Agent/Tool) MUST warn about dependencies (used in Crew/Flow) or use Soft Delete.
6.  **Contract-First UI:** All Lists, Forms, and SSE streams MUST have pre-defined Zod schemas.
7.  **Fallback Resilience:** If the primary LLM fails, the system MUST retry with a fallback model defined in the Router config.
8.  **Asset Integrity:** Knowledge sources are versioned; re-indexing does not break existing citations until confirmed.
9.  **Availability Workspace:** All workspace-scoped entities (Agent, Crew, Pattern, Template, Automation, Service, Tool, Prompt Archetype) MUST have `availability_workspace: List<Enum>` field (Global, Discovery, Design, Delivery, Growth & Market, Product Management).
10. **Cost Estimation:** Agent/Crew briefing MUST display estimated token cost before execution begins — user approves or modifies the plan.

## 4. System Architecture & Constraints
*   **Architecture Pattern:** Modular Monolith with Vertical Slice Architecture.
    *   **Backend:** Python 3.12+, FastAPI (Async), **crewAI** (Orchestration), **LangChain** (Infrastructure/RAG).
    *   **Task Queue:** **Inngest** (Durable Execution for Crews and Ingestion).
    *   **Observability:** **Langfuse** (Tracing, Cost tracking, Prompt Management).
*   **Frontend Stack:** Next.js 16 (App Router), Tailwind, Shadcn/UI, React Flow.
*   **Database:** Supabase (PostgreSQL + **pgvector**).
    *   **Vector Dims:** 768 dimensions (default) or 1536 (OpenAI compatible).
*   **Communication:**
    *   **BFF:** Next.js Route Handlers proxy to FastAPI.
    *   **Streaming:** SSE (Server-Sent Events) for Agent thinking/outputs.

# Use case: Aplikacja do zamawiania zdrowych posiłków

<aside>

Wyobraźmy sobie, że projektujemy aplikację, która umożliwia szybkie zamawianie zdrowych posiłków. Jak w takim przypadku zastosujemy **Jobs to be Done (JTBD)** i **Personę**?

</aside>

### **Jobs to be Done (JTBD)** – fokus na zadaniu

<aside>

📌 **Przykładowy Job**:

*"Chcę zamówić zdrowy posiłek do biura w mniej niż 5 minut, żeby uniknąć niezdrowych przekąsek i skupić się na pracy."*

✅ **Co wynika z JTBD?**

- Kluczowa jest szybkość zamówienia.
- Użytkownik nie chce spędzać czasu na przeglądaniu długiej listy opcji.
- Zdrowe jedzenie jest ważne, ale musi być dostępne od ręki.
- Można zaproponować opcję „szybkie zamówienie” na podstawie wcześniejszych wyborów.

</aside>

### **Persona** – fokus na użytkowniku

<aside>

👩 **Przykładowa Persona:** **Anna, 32 lata, Marketing Manager**

- **Demografia**: Mieszka w dużym mieście, pracuje w korporacji, zarabia dobrze.
- **Zachowanie**: Często zamawia jedzenie do biura, bo nie ma czasu gotować.
- **Motywacja**: Chce dbać o zdrowie, ale jednocześnie nie chce tracić czasu na gotowanie czy długie przeglądanie menu.
- **Ból (pain point)**: Dostawy często są zbyt wolne lub menu jest za bardzo skomplikowane.

✅ **Co wynika z Persony?**

- Aplikacja powinna oferować opcje szybkich zamówień dla zapracowanych profesjonalistów.
- Komunikacja marketingowa powinna podkreślać oszczędność czasu i zdrowe wybory.
- Można dodać funkcję subskrypcji na „codzienne zdrowe zestawy” dostosowane do stylu życia Anny.

</aside>

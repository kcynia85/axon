# Strony i treści zastępcze

### *Empty states*

<aside>
To nie komunikaty o błędach. Dobrze aby taka strona zawierała:

- ikonę
- nagłówek
- krótki akapit tekstu z instrukcją co dalej
- CTA
</aside>

<aside>
Lepiej zapobiegać niż leczyć. Dlatego można zaprojektować onboarding z przykładowymi treściami. Uczenie przez działanie wypłaszcza krzywą uczenia.

</aside>

### Puste wyniki wyszukiwania

<aside>
**Jeśli nie ma filtrów**

- Zasugerowanie zmiany frazy przez podanie
    - podobnych fraz
    - najczęściej wyszukiwanych fraz
- Zmiana kategorii
    - podobne kategorie
    - najpopularniejsze kategorie
- Elastyczny system pokazujący najbliższe frazy (Google)
- Opcja zaawansowanej wyszukiwarki
</aside>

### Brak gotowości systemu do wyświetlenia danych

<aside>

**Pomocne zwroty jeśli nie wiemy**

*“Jeszcze nie możemy pokazać…”*

“Na razie tu nic nie ma…”

“Wciąż przetwarzamy…”

**Pomocne zwroty jeśli wiemy** 

Przyczyny:

- Brak dostępu do sieci
- Problem z ładowaniem dawnych
- Nieaktualna wersja aplikacji

Podpowiedzi dla użytkowników:

*“Przeładuj stronę”*

*“Wyloguj się i zaloguj ponownie”*

*“Wyczyść cache”*

</aside>

### Dobre praktyki

Inspiracja → Empty states Shopify 

<aside>

- Strategia zamiast reaktywności
- Sedno tego co chcemy przekazać, unikając lakoniczności
- Dostosowanie komunikatu do rynku. Różnica pomiędzy rynkiem amerykańskim, a nordyckim
- Dbałość o szczegóły. Atrakcyjne i dopasowane ilustracje mogą studzić negatywne emocje. Mogą również wpłynąć pozytywnie na zwiększenie zaangażowania
- Spójność:
    - Wizualna
    - Tonu głosu
    - Gramatyki
    - Dobór słownictwa
- Dbałość o emocje odbiorców. Unikanie smutnych buziek.
</aside>

# Określanie potrzeb i problemów użytkowników (+Use Cases)

<aside>

Na podstawie zebranych danych określono kluczowe potrzeby:

- **Potrzeba uproszczonej nawigacji:** Użytkownicy chcą krótszego, intuicyjnego procesu zamówienia.
- **Potrzeba jasnych informacji o statusie zamówienia i promocjach:** Wielu użytkowników rezygnuje z zamówienia, jeśli nie widzi informacji o promocjach lub statusu płatności.
- **Potrzeba zróżnicowanych metod płatności:** Brak możliwości zapłaty przez szybkie płatności zniechęca użytkowników.

</aside>

# Gamification & Behavioral Audit Checklist

> **Cel:** Szybka diagnoza stanu behawioralnego produktu.
> **Użycie:** Przejdź przez pytania dla kluczowych flow użytkownika.

## 1. The Hook (Pętla Nawyku)

### Trigger (Wyzwalacz)
- [ ] Czy zdefiniowano *Wewnętrzny Trigger* (emocję/ból), który napędza użycie? (np. Nuda, Lęk).
- [ ] Czy *Zewnętrzne Triggery* (powiadomienia, maile) są skorelowane z momentem, gdy użytkownik może podjąć akcję?
- [ ] Czy powiadomienia zawierają jasne CTA (Call to Action)?
- [ ] Czy unikamy spamu triggerami (czy każdy trigger niesie wartość)?

### Action (Akcja & Bariery)
- [ ] Czy kluczowa akcja jest fizycznie najprostszą rzeczą do zrobienia na ekranie?
- [ ] **Test 6 Barier:**
    - [ ] Czy czas wykonania jest minimalny?
    - [ ] Czy koszt (pieniężny) jest jasny i akceptowalny?
    - [ ] Czy wysiłek fizyczny (kliknięcia) jest zredukowany?
    - [ ] Czy wysiłek umysłowy (konieczność myślenia) jest zredukowany?
    - [ ] Czy akcja jest społecznie akceptowalna?
    - [ ] Czy pasuje do rutyny użytkownika?

### Variable Reward (Zmienna Nagroda)
- [ ] Czy użytkownik otrzymuje nagrodę *natychmiast* po akcji?
- [ ] Czy nagroda ma element zmienności/nieprzewidywalności?
- [ ] Czy stosujemy miks nagród?
    - [ ] **Tribe:** Uznanie społeczne, rywalizacja.
    - [ ] **Hunt:** Zasoby, informacje, okazje.
    - [ ] **Self:** Mistrzostwo, domknięcie zadania.

### Investment (Inwestycja)
- [ ] Czy akcja użytkownika "ładuje" kolejny trigger? (np. wysłanie wiadomości -> oczekiwanie).
- [ ] Czy produkt staje się lepszy wraz z użyciem (więcej danych, lepsze algorytmy)?
- [ ] Czy użytkownik personalizuje produkt (Ikea Effect)?

---

## 2. Octalysis Check (Motywacja)

### White Hat vs Black Hat
- [ ] Czy system buduje długofalowe zaangażowanie (White Hat: Rozwój, Znaczenie)?
- [ ] Czy nie nadużywamy technik stresowych (Black Hat: FOMO, Strata) w sposób prowadzący do wypalenia?

### Core Drives Balance
- [ ] **Meaning:** Czy użytkownik czuje, że robi coś ważnego?
- [ ] **Accomplishment:** Czy widać postęp (Paski, Levele)?
- [ ] **Empowerment:** Czy użytkownik ma wybór i może testować strategie?
- [ ] **Ownership:** Czy użytkownik czuje, że coś posiada (Kolekcje, Profil)?
- [ ] **Social:** Czy jest element społeczny (nawet pasywny)?
- [ ] **Scarcity:** Czy coś jest limitowane/ekskluzywne?
- [ ] **Unpredictability:** Czy jest element ciekawości/tajemnicy?
- [ ] **Loss:** Czy użytkownik boi się coś stracić (np. Streak)?

---

## 3. Flow State & UI
- [ ] **Jasność celu:** Czy na każdym ekranie jest *jeden* główny cel?
- [ ] **Feedback:** Czy każda interakcja ma wizualną/dźwiękową odpowiedź?
- [ ] **Koncentracja:** Czy w krytycznych momentach usunięto zbędną nawigację?
- [ ] **Balans:** Czy poziom trudności rośnie wraz z umiejętnościami użytkownika (Scaffolding)?

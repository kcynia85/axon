# Create PRP (Best Practices)

## Purpose
Generate a complete PRP for a feature based on the Tech PRD.  
Follow this guide to ensure context, planning, tasks, and validation gates are fully included.

## Workflow
1. **Read Tech PRD**
   - Understand feature requirements, business logic, and constraints.
   - Identify entities, actions, invariants, and domain vocabulary.

2. **Use PRP Template**
   - Fill the PRP Template with feature-specific details.
   - Include:
     - Metadata
     - Entities & Domain Vocabulary
     - Invariants
     - Architecture & Constraints
     - DB/Data requirements
     - Sync/Async operations
     - Caching rules
     - Domain behaviors
     - Implementation steps

3. **Research**
   - Explore codebase for similar features or patterns.
   - Collect references, code snippets, documentation links.
   - Document gotchas or library quirks.

4. **Generate Output**
   - Save filled PRP as `PRPs/{feature-name}.md`.
   - Generate `IMPLEMENTATION.md` using `templates/implementation-template.md`.
   - Ensure `IMPLEMENTATION.md` is reviewed and approved before implementation.

## Chain of Thought (CoT) Rule
**Application:** When defining implementation steps in a PRP, avoid simple instructions like "Write function X".
**Requirement:** Instead, use: "Analyze potential edge cases, plan the type structure, and ONLY THEN write the code."
**Benefit:** This drastically increases the quality of the code generated by the LLM by reducing logical errors.

# Refactor Roadmap Planner Skill

This skill analyzes a codebase and creates a long-term refactoring roadmap.

## Goals

- identify architectural problems
- prioritize improvements
- reduce technical debt

## Analysis Areas

Analyze:

- module complexity
- duplicated logic
- architectural inconsistencies
- testing gaps
- dependency coupling

## Workflow

1. Scan repository structure.
2. Detect complex or large modules.
3. Identify duplicated patterns.
4. Evaluate test coverage and quality.
5. Identify architectural weaknesses.
6. Create a prioritized refactoring roadmap.

## Rules

- prioritize high-impact improvements
- avoid risky large refactors
- propose incremental changes
- focus on maintainability

## Output

Produce a roadmap including:

- detected technical debt
- prioritized refactoring tasks
- estimated complexity
- recommended implementation order

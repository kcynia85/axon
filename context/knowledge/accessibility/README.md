# ♿ Accessibility Standards (A11y)

Ten katalog zawiera modułowe zasady dostępności cyfrowej dla projektu. 
W zależności od realizowanego zadania, odwołaj się do odpowiedniego pliku:

## 📂 Modules Map

### 1. [semantics.md](./semantics.md)
**Kiedy używać:** Tworzenie layoutu, struktura HTML, nagłówki.
- Semantyczne znaczniki (`header`, `main`, `nav`).
- Hierarchia nagłówków (`h1`-`h6`).
- Listy i podstawowe role ARIA.

### 2. [visuals.md](./visuals.md)
**Kiedy używać:** Stylowanie CSS, dobór kolorów, typografia.
- UI Clarity (ikony, etykiety).
- Ostrzeżenia o miganiu (Motion & Flashing).
- *(Wkrótce: Kontrast i Typografia)*

### 3. [interaction.md](./interaction.md)
**Kiedy używać:** Implementacja nawigacji klawiaturą i focusa.
- Visible Focus (`outline`).
- Linki i przyciski.
- Pełna dostępność z klawiatury.

### 4. [forms.md](./forms.md)
**Kiedy używać:** Tworzenie formularzy i walidacji.
- Etykiety (`label`) i `placeholder`.
- *(Wkrótce: Walidacja i obsługa błędów)*

### 5. [media.md](./media.md)
**Kiedy używać:** Dodawanie obrazów, wideo i audio.
- Tekst alternatywny (`alt`) dla obrazów i wykresów.
- Napisy i transkrypcje audio/wideo.
- Kontrolki odtwarzacza (brak autoplay, obsługa z klawiatury).

### 6. [data.md](./data.md)
**Kiedy używać:** Prezentacja danych tabelarycznych.
- Struktura tabel (`th`, `scope`).

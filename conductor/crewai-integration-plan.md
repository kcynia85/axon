# CrewAI Integration Plan: Port & Adapter for Agent and Crew Nodes

## 🎯 Objective
Replace the current stubbed `CrewRunnerFacade` with a fully functional CrewAI execution engine. This engine will act as an adapter in a Hexagonal Architecture (Ports & Adapters), executing `Agent` and `Crew` entities generated by the Studio and Meta-Agent directly on the Space Canvas.

## 🏗️ Architecture & Boundaries (Hexagonal & Modular Monolith)

Following the `architecture-planner` and `python-fastapi-expert` guidelines, we will strictly separate business logic (Domain) from the external CrewAI framework (Infrastructure).

### 1. Domain Layer (`app/modules/agents/domain/ports/crew_engine.py`)
Define abstract interfaces (Ports) that the Application layer will depend on. The core domain should not know about CrewAI.

```python
from abc import ABC, abstractmethod
from typing import Dict, Any, List
from app.modules.agents.domain.models import AgentConfig
from app.modules.workspaces.domain.models import Crew

class ICrewEngine(ABC):
    @abstractmethod
    async def kickoff_crew(self, crew_config: Crew, hydrated_agents: List[AgentConfig], user_input: str) -> Dict[str, Any]:
        pass
        
    @abstractmethod
    async def run_single_agent(self, agent_config: AgentConfig, user_input: str) -> Dict[str, Any]:
        pass
```

### 2. Infrastructure Layer (`app/modules/agents/infrastructure/crewai/adapter.py`)
Implement the CrewAI Adapter. This is the **only** place where `import crewai` will be allowed.

*   **Agent Mapping**: Transform Axon's `AgentConfig` into CrewAI's `Agent` object.
    *   `agent_role_text` -> `role`
    *   `agent_goal` -> `goal`
    *   `agent_backstory` -> `backstory`
    *   Map Axon `Tool` references to actual Python callables for CrewAI tools.
*   **Task Mapping**: 
    *   Read `tasks` from the Crew's `metadata` (populated by Studio's `SequentialTaskFormSchema`).
    *   Transform them into CrewAI `Task` objects, linking them to the matched CrewAI `Agent`.
*   **Crew Mapping**: Transform Axon's `Crew` into CrewAI's `Crew` object.
    *   Map `crew_process_type` (`Sequential`, `Hierarchical`) to `crewai.Process`.
    *   If Hierarchical, assign the `manager_agent_id` as the manager.

## ⚙️ Space Canvas Execution Flow
1. **Trigger**: User clicks "Run" on a Crew Node or Agent Node in the Space Canvas.
2. **Endpoint**: Frontend calls `POST /spaces/{space_id}/nodes/{node_id}/run`.
3. **Inngest Workflow**: The router emits an Inngest event (e.g., `crew/execution.requested`).
4. **Adapter Execution**: The Inngest worker picks up the job, invokes `CrewRunnerUseCase`, and the CrewAI Adapter starts the execution (`crew.kickoff()`).
5. **Streaming/Feedback**: The Adapter uses CrewAI's step callbacks (`step_callback`) to emit Inngest events (`system.entity.progress_updated`). The frontend listens to these to update the Canvas Node progress bar (`progressValue`) and status (`isWorking`, `isDone`).

## 📋 Implementation Steps

1. **Install Dependencies**: `uv add crewai langchain-openai` (or appropriate LLM providers).
2. **Define Ports**: Create `ICrewEngine` interface in the domain layer.
3. **Build Adapter**: Implement `CrewAIAdapter` inside `infrastructure/crewai`. Implement mapping logic.
4. **Tool Registry Integration**: Expose Axon native tools (Knowledge Search, MCPs) as Langchain/CrewAI compatible `BaseTool` instances inside the Adapter.
5. **Refactor Application**: Update `crew_runner.py` to use the injected adapter instead of the fake sequential loop.
6. **Update Studio/Meta-Agent Schemas**: Ensure `tasks` definitions generated by Meta-Agent and Studio are saved correctly into `Crew.metadata` so the adapter can read them.
7. **Canvas State Sync**: Wire up the CrewAI callbacks to update the Space Canvas node states in real-time (progress percentages, active agent highlighting).

## 🛡️ Best Practices Applied
- **Modular Monolith**: The CrewAI logic is encapsulated in the `agents` infrastructure module.
- **Dependency Injection**: FastAPI routers and Inngest handlers will inject the adapter, making testing easy.
- **Functional Core**: Business rules remain pure, while CrewAI handles the LLM state machine.
